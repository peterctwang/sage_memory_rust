> Rust workspace implementing the **SAGE** self-evolving graph-memory engine
> (paper `SAGE_Paper.md` / arXiv 2605.12061) as a long-term memory layer
> for LLM agents.

This is **not** a research-quality port of the paper; it is a working Rust
baseline you can run today. The full GFM neural net + GRPO training loop
remain multi-week work, but every architectural surface is in place and
covered by tests. See [SPEC_SAGE_Rust.md](SPEC_SAGE_Rust.md) for the design
contract and [CONSTITUTION.md](CONSTITUTION.md) for the development rules.

---

## What's working today

- **End-to-end pipeline**: real natural-language documents → LLM-extracted
  triples → persistent graph → semantic retrieval, all driven by 7 CLI
  subcommands.
- **Three LLM backends**:
  - `MockLlm` — deterministic FIFO scripted responses (default, no auth)
  - `AnthropicLlm` — Messages API + exponential-backoff retry (feature `anthropic`)
  - `ClaudeCliLlm` — local `claude` binary subprocess (feature `claude-cli`,
    no API key needed — uses your existing Claude Code subscription)
- **Persistence**: `MemGraphStore` (in-process) and `SledGraphStore` (disk).
- **Retrieval acceleration**: HNSW vector index auto-wired via `SageEngine`.
- **Eval**: Recall@k / Precision@k / F1@k / MRR with `EvalRunner`.
- **GFM single-layer forward pass** (M3 spike): vector gate + PReLU + LN +
  message-passing via custom `scatter_add` CPU kernel, ready for a tensor
  backend.
- **GRPO numerical primitives**: group-relative advantage, clipped ratio,
  surrogate objective.
- **Multi-tenant isolation** baked into every trait.

## Quick start

```bash
# Build + run the canned in-memory demo
cargo run -p sage-cli -- demo

# Real LLM pipeline against a persistent sled graph
export SAGE_CLAUDE_BIN="$(which claude)"   # Windows: claude.cmd
cargo run -p sage-cli -- ingest \
    --db graph.sled --doc-id 1 \
    --doc "Alice Liddell founded Acme Industries in 2023 in Boston."

cargo run -p sage-cli -- query --db graph.sled --k 3 "Who founded Acme"
cargo run -p sage-cli -- eval --db graph.sled --k 5 < samples.json
```

Batch ingest from JSONL (`{"doc_id": N, "text": "..."}` per line):

```bash
cargo run -p sage-cli -- ingest-batch --db graph.sled --jsonl docs.jsonl
```

## Workspace layout

| crate | role |
|---|---|
| `sage-core` | shared types, traits (`GraphStore`, `Reader`, `Embedder`, `VectorIndex`), reward math, `scatter_add` kernels |
| `sage-graph` | `MemGraphStore` + `SledGraphStore` (feature `sled`, default-on) |
| `sage-embed` | `DeterministicEmbedder` + `HnswIndex` (feature `hnsw`, default-on) |
| `sage-llm` | `LlmClient` trait + `MockLlm` + `AnthropicLlm` + `ClaudeCliLlm` |
| `sage-writer` | `LlmWriterPolicy`, `TripleSanitizer`, `apply_action_embedded` |
| `sage-reader` | `HeuristicReader` (soft addressing 6/6 stimulus), `GfmLayer` |
| `sage-eval` | metrics + `EvalRunner` + reward loop + GRPO primitives |
| `sage-runtime` | `SageEngine` — high-level orchestration |
| `sage-cli` | `sage` binary — 8 subcommands |
| `tests-support` | shared fixtures, proptest strategies, contract suites |

## What is **not** done

Honest scope boundary (see [`SPEC_SAGE_Rust.md` §13](SPEC_SAGE_Rust.md)):

| missing | blocked on |
|---|---|
| Multi-layer GFM + Context/Schema head | candle integration, multi-day spike |
| GRPO trainer (real LLM weight updates) | GFM above + LoRA + GPU |
| BGE/E5 production embedder | tokenizer + ONNX/candle model loading |
| Full self-evolution loop (`engine.evolve(cfg)`) | all of the above |

The architecture is laid out so these can drop in as the *implementations*
of existing traits without restructuring downstream code.
